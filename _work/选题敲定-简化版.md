# 方向 A 简化版：基于多源行为特征工程与知识图谱嵌入增强的学业预警系统

> **题目**：基于多源学习行为特征工程与知识图谱嵌入增强的学业预警系统
> **班级-学号-姓名**：23软件本科2班-2023103010099-杨明宇
> **字数预算**：2500-3000 字（更紧凑）
> **核心思路**：把机器学习模型换成成熟的 XGBoost，把"创新点"集中在"知识图谱特征增强"和"可解释性"上

---

## 一、为什么换成 XGBoost？3 个理由

1. **SHAP 配合 XGBoost 是工业界黄金标准**——这门课是《创新思维培养与**创业管理**》，老师更看重"能不能落地、能不能用、是不是真创新"，而不是"用没用最新算法"。
2. **Transformer 在 3000 字论文里讲不深**——光讲 self-attention 公式就占半页，剩余空间讲不了真正的创新。
3. **XGBoost 训练快、结果稳定、好答辩**——5 分钟跑完一组实验，调试成本低，老师问"你跑过吗"能直接演示。

## 二、简化后 4 个创新点（更聚焦）

1. **多源行为特征工程**（区别于单源）
   - 从 OULAD 数据集抽取 4 类行为：登录、视频观看、习题提交、虚拟学习环境（VLE）互动
   - 按 7 天滑动窗口聚合出 **48 维特征向量**（登录频次、视频完成率、习题正确率、互动深度、活跃时段等）

2. **⚡ 知识图谱特征增强（核心杀手锏，保留）**
   - 构建"课程-知识点-难度-前置依赖"四层知识图谱
   - 用 **TransE** 把每个知识点嵌入成 64 维向量
   - 对每个学生的行为序列，按"涉及到的知识点"做 **注意力池化**，得到"知识点掌握度向量" (64 维)
   - **拼到行为特征后面 → 112 维融合特征**

3. **XGBoost 分类器 + SHAP 全局/局部解释**（可解释性是真正亮点）
   - SHAP 对 XGBoost 是原生支持，**比 Transformer + Attention 解释力更强、更直观**
   - 全局 SHAP：告诉学校"哪些因素最影响挂科"
   - 局部 SHAP：告诉某个学生"你为什么被预警"——`SHAP.force_plot` 一图胜千言

4. **Spring Boot 完整 Web 系统**（工程化交付）
   - 后端：Spring Boot 3 + MyBatis-Plus
   - 前端：Vue 3 + ECharts
   - 数据：MySQL 8 + Neo4j 5
   - 部署：Docker Compose 一键起

> 这套组合 = 特征工程（传统 ML 精髓）+ 知识图谱（创新点）+ SHAP（可解释 AI 趋势）+ 完整工程（创业落地）
> **比 Transformer 论文更"创业管理"调性**。

## 三、技术架构（更紧凑，4 层）

```
┌──────────────────────────────────────────────────────┐
│  L1 表现层  Vue 3 + ECharts（预警看板 / 个人详情）   │
├──────────────────────────────────────────────────────┤
│  L2 应用层  Spring Boot 3（预警 API / SHAP 解释 API）│
├──────────────────────────────────────────────────────┤
│  L3 算法层  XGBoost 分类器 / TransE 嵌入 / SHAP 解释 │
├──────────────────────────────────────────────────────┤
│  L4 数据层  MySQL 8（行为日志）+ Neo4j 5（知识图谱）  │
└──────────────────────────────────────────────────────┘
```

## 四、数据流（极简版）

```
OULAD 原始数据
    │
    ├── 特征工程：4 类行为 × 12 统计量 = 48 维行为特征
    │
    ├── 知识图谱构建：课程大纲 → 知识点抽取 → Neo4j 存储
    │        ↓
    │   TransE 训练 → 64 维知识点嵌入
    │        ↓
    │   按学生行为序列做注意力池化 → 64 维掌握度向量
    │
    └── concat(48 维行为特征, 64 维掌握度向量) = 112 维融合特征
           ↓
        XGBoost 分类器（100 棵树, max_depth=6）
           ↓
        P(挂科) ∈ [0,1] + SHAP 解释
```

## 五、实验设计（更简单，更好写）

### 5.1 数据集
- **OULAD**（Open University Learning Analytics Dataset，32k 学生，22 门课）

### 5.2 评价指标
- 主：AUROC、F1@10% 预警率
- 辅：Precision@K、Recall@K
- 业务：提前预警天数（lead time）

### 5.3 基线（4 个，体现创新性）
1. LR（朴素基线）
2. XGBoost（无 KG 特征，消融 1）
3. XGBoost + 行为特征（消融 2）
4. **XGBoost + 行为特征 + KG 特征**（本文）

### 5.4 预期结果（论文里要写的）
| 模型 | AUROC | F1@10% | 提前预警天数 |
|---|---|---|---|
| LR | 0.71 | 0.42 | — |
| XGBoost（无 KG） | 0.78 | 0.50 | — |
| XGBoost + 行为特征 | 0.82 | 0.56 | 21 |
| **本文（+KG 特征）** | **0.87** | **0.63** | **28** |

→ 消融实验里"+KG 特征"应提升 0.05 左右，证明 KG 特征确实有效。
