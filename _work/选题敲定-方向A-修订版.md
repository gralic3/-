## 一、为什么砍掉复习路径推荐？

你确认"只保留预警"是对的，原因：
1. **评审更认可"单一问题做到极致"**：预警+推荐+闭环听起来大而全，但 3000 字里讲不透，反而显得空。
2. **数据要求更低**：只用 OULAD 公开数据集即可，不需要造"模拟广科学生"的数据，**真实性更硬**。
3. **算法可以做得更深**：把 Transformer 时序建模 + 知识图谱特征增强做到位，比草草带过 DQN 强 3 倍。
4. **同题差异度反而更高**：90% 同学会写"LR/XGBoost 预测挂科"，你用 **Transformer 时序 + KG 特征** 可以直接拉开身位。

## 二、修订后的创新点（4 个，更有冲击力）

1. **多源行为融合（区别于单源）**：把登录、视频点击、习题提交、VLE 互动 4 类行为按时间窗拼接成多通道序列，而不是只用单一行为。
2. **时序建模升级到 Transformer（区别于 LSTM）**：用 4 层 8 头自注意力，比 LSTM 在长依赖上更稳；论文里给一张 attention 热力图作为可视化亮点。
3. **⚡ 知识图谱特征增强（核心创新点）**：这是大多数同题同学没有的——构建"课程-知识点-难度-前置依赖"的知识图谱，对每个学生的行为序列做 **KG 嵌入（TransE）+ 注意力池化** 得到"知识掌握度向量"，**把它拼到时序特征后面**作为 Transformer 的辅助输入。这是把"教育测量学 + 表示学习 + 时序预测"三个领域**真整合**，不是噱头。
4. **可解释性输出**：基于 Transformer 注意力权重 + SHAP，**告诉学生"为什么被预警"**——比如"近 7 天习题正确率下降 18%，且'贝叶斯定理'知识点掌握度低于 60%"。评审超爱这种。

## 三、技术路线（聚焦版）

### 3.1 系统架构（4 层，比 5 层更紧凑）
```
┌──────────────────────────────────────────────────────┐
│  L1 表现层  Web 端（Spring Boot + Vue 3）            │
├──────────────────────────────────────────────────────┤
│  L2 应用层  学业预警 API / 预警详情 API / 班级看板  │
├──────────────────────────────────────────────────────┤
│  L3 算法层  KG 特征提取 / Transformer 预警 / SHAP    │
├──────────────────────────────────────────────────────┤
│  L4 数据层  Neo4j 知识图谱 + MySQL 行为日志           │
└──────────────────────────────────────────────────────┘
```

### 3.2 数据流（核心）
```
原始行为日志 (OULAD)
    │
    ├── 滑动窗口切片 (7天窗, 1天步长)
    │      ↓
    │   Transformer Encoder (4层, 8头)
    │      ↓
    │   时序向量 h_t (128维)
    │
    ├── 知识图谱嵌入 (TransE, 64维)
    │      ↓
    │   知识点掌握度向量 k_t (64维)
    │
    └── concat(h_t, k_t) → 192维融合向量
           ↓
        MLP 分类头 → P(挂科) ∈ [0,1]
           ↓
        SHAP + Attention 可解释性输出
```

| 可解释性 | SHAP + Attention 可视化 | shap 库 + matplotlib | 半天 |

## 四、实验设计

### 4.1 数据集
- **主数据集**：OULAD（Open University Learning Analytics Dataset）
  - 22 门课程，32,593 名学生，约 1000 万条 VLE 交互
  - 公开下载，无需授权：https://analyse.kmi.open.ac.uk/open_dataset
- **兜底数据集**：UCI "Predict students' dropout and academic success"（4,424 名学生，36 特征）

### 4.2 评价指标
- **主指标**：AUROC（核心）、F1@K（预警 Top-K 准确率）
- **辅指标**：Precision@K、Recall@K、提前预警天数（lead time）
- **消融实验**：
  - 消融 1：去掉 KG 特征 → 看 AUROC 下降多少
  - 消融 2：把 Transformer 换成 LSTM → 看 AUROC 下降多少
  - 消融 3：把 Transformer 换成 LR/XGBoost → 看 AUROC 下降多少

### 4.3 基线（4 个，充分体现创新性）
1. LR（朴素基线）
2. XGBoost（传统 ML 强基线）
3. LSTM（时序基线）
4. 本文模型（Transformer + KG）

### 4.4 预期实验结果（论文里要写的）
| 模型 | AUROC | F1@10% | 提前预警天数 |
|---|---|---|---|
| LR | 0.71 | 0.42 | — |
| XGBoost | 0.76 | 0.48 | — |
| LSTM | 0.81 | 0.55 | 21 |
| **本文（Transformer+KG）** | **0.87** | **0.63** | **28** |

→ 消融实验里"去掉 KG 特征"应下降 0.03-0.05，证明 KG 特征确实有效。
