# 方向 A 敲定：基于学习行为时序分析与知识图谱的学业预警及个性化复习路径推荐

## 一、痛点（问题分析与背景研究，20%）

### 1.1 真实问题
- **预警滞后**：传统"挂科预警"通常在期中/期末出分后才触发，错失干预窗口。
- **干预手段单一**：现有教务系统只输出"可能挂科"标签，不告诉学生**该复习什么、按什么顺序复习**。
- **资源过载**：MOOC、B 站、知乎、教材习题浩如烟海，学生不知道哪一份资料对自己当前知识薄弱点最有效。
- **冷启动**：转专业、复学、新生等无历史行为数据的用户，传统协同过滤直接失效。

### 1.2 现有方法局限性（文献综述时要打的靶子）
| 现有方法 | 局限 |
|---|---|
| 朴素成绩预测（LR / 决策树） | 只用静态特征（性别、专业、籍贯），忽略行为时序 |
| 协同过滤推荐 | 冷启动严重；只能推荐"别人也看过"，无法推荐"我现在该看的" |
| 通用知识图谱推荐（如 DKN） | 不结合学习者的**当前薄弱知识点**和**剩余时间**约束 |
| LSTM 学业预警 | 通常只到"预警"为止，**不延伸到复习路径生成**，半截子工程 |
| LLM 直接对话式辅导 | 缺乏结构化知识约束，容易"一本正经地胡说"；且不能批量服务 |

### 1.3 本文切入点（差异化创新点 ⚡）
> **把"预警"和"推荐"做成一个端到端闭环，而不是两个独立模块。**

具体说：
1. **多源行为时序建模**：把登录、点击、答题、错题本、讨论区发帖、图书馆门禁、自习室预约 7 类行为按时间窗切片，用 **Transformer 时序分类器**预测"未来 4 周挂科概率 P(fail)"（解决"预警滞后"）。
2. **细粒度知识图谱**：构建"课程-章节-知识点-前置知识点-资源（视频/习题/笔记）"五层图，每个知识点带"掌握度"标签。
3. **基于薄弱点的强化学习复习路径推荐**：把"复习路径"建模为 MDP，状态=当前知识点掌握度+剩余天数+学习风格，动作=选下一学习资源，奖励=知识掌握度提升×时间效率，**用 DQN 训练策略网络**（这是真正跨领域——把推荐系统 + RL + 教育心理学结合）。
4. **冷启动兜底**：用 LLM 把课程大纲+教材目录自动结构化成知识图谱初始节点，新生/转专业用户 0 数据也能用。
5. **反馈闭环**：每次学习后用"答题正确率+耗时"实时更新掌握度，反哺 RL 策略。

> 这套组合拳 = 时序预测（预警）+ 知识图谱（结构化知识）+ DQN（路径优化）+ LLM（冷启动），
> 既满足"创新性 30%"，又满足"技术难度 25%"，还能"解决方案 20%"讲圆——性价比最高。

## 二、技术路线图（创新解决方案设计 30% + 技术实施 30%）

### 2.1 系统架构（5 层）
```
┌──────────────────────────────────────────────────────┐
│  L1 表现层  Web 端（Spring Boot + Vue）+ 微信小程序   │
├──────────────────────────────────────────────────────┤
│  L2 应用层  学业预警 API / 复习路径 API / 知识图谱浏览 │
├──────────────────────────────────────────────────────┤
│  L3 算法层  Transformer 预警 / DQN 路径推荐 / KG 推理  │
├──────────────────────────────────────────────────────┤
│  L4 数据层  Neo4j 知识图谱 + MySQL 行为日志 + Redis   │
├──────────────────────────────────────────────────────┤
│  L5 数据采集层  LMS 埋点 + 图书馆 API + 一卡通门禁    │
└──────────────────────────────────────────────────────┘
```

### 2.2 关键算法
| 模块 | 算法 | 创新点 |
|---|---|---|
| 行为时序切片 | 滑动窗口 (窗口=7 天, 步长=1 天) | 多源行为向量化拼接 |
| 挂科预测 | Transformer Encoder (4 层, 8 头) | 比 LSTM 在长依赖上更稳 |
| 知识图谱构建 | 课程大纲 + LLM 抽取 + 专家校验 | 三方共建，比纯人工/纯自动都准 |
| 掌握度估计 | IRT (项目反应理论) + BKT (贝叶斯知识追踪) 混合 | 经典教育测量模型，论文好引用 |
| 复习路径生成 | DQN (ε-greedy, 经验回放) | 状态含"剩余天数"硬约束 |
| 冷启动 | LLM (Qwen2.5-7B) 结构化抽取 | 离线批处理，推理成本可控 |

### 2.3 实验设计
- **数据集**：公开 OULAD（Open University Learning Analytics Dataset，22 门课、32k 学生）+ 模拟广东科技学院某专业 200 学生一学期行为（论文里要标明"部分数据为模拟"）
- **基线**：LR / XGBoost / LSTM 预警；协同过滤 / DKN 推荐
- **指标**：
  - 预警：AUROC、F1@K（提前 4 周预测）
  - 推荐：路径完成率、平均知识掌握度提升、用户停留时长
  - 系统：API P99 延迟 < 200ms

### 2.4 评估结论（论文里要给出的预期结果）
- 预警 AUROC 0.87+，比 LR 基线 +12%
- 复习路径完成率比协同过滤 +35%
- 冷启动用户首日推荐可用率 100%
